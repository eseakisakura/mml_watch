　　■ PowerShell / FM Synthesis Chip - Tone Editor 


	当テキストは、構造化エディタで組まれてます。

	-- スクリプト側も構造化エディタに対応しています。--

 
　　□ 概要 


	FM音源の音色エディタです。
	FM合成波形を視覚的に捉えられるようになってます。

	レジスタ値や音色マトリクス指定を、マウスドラッグで
	変更できるようにしたものです。

	"VRC7", "OPL", "OPN", "OPM" の各音色マトリクスに対応し、変更のち、
	すぐに試聴できるのが特徴です。

 	
　　□ 注意点について 


	・"ppmck", "NSDlib" の、レジスタ記載による
	vrc7音色を出力する場合、メニューの切り換えが必要です。

	・"mucom", "fmp7", "mxdrv" 形式の、マトリクス音色出力を選択できますが、
	内部的には "PMD" 出力となっています。

	・FM Operator ウィンドウの、ADSRエンベロープ表示は、
	視覚的な減衰率を捉えたものとなっています。


	・MaskとSSG-EG機能は、"opn", "opm"共用となっています。


	・SSG-EGのエンベロープは、固定表示となってます。


	・FM合成のサイン波表示は、出力波形との擦り合わせを行ったため、
	近似値は出てますが、完全な再現を目指したものではありません。

	・サイン波形のグラフ表示は、MLの倍数値が高いとある程度、誤差が出ます。
	三角関数多用による演算負荷を軽減するため、float処理しているためです。

	・Multipleが 0.5倍のときは、
	前半の波形のみ表示し、後半の波形は表示されません。


	・もし起動できない場合、
	scriptフォルダの、"fm_editor.xml" 環境ファイルを削除して下さい。
	初期設定と同じになります。


 
　　□ 基本設定 - クイックスタート 


	まず、環境設定から始めます。

　1.	fm_editorを起動し、オプションから settingをクリック、
	環境設定が立ち上がります。


　2.	各リストボックスに、必要となるファイル、プログラムを登録します。

	"binary", "player", "editor", "dos"タブに、D&Dで順に登録していきます。
	(PMDを64bitOSでの使用の場合、x64にチェックを入れて下さい。)


　3.	再生ボタンを押して、FM変調波の音が出れば、基本的な設定は完了です。
	(もし音が出ない場合、メニューから、再生できるプレイヤーの選択をしてみて下さい。)


 
　　□ 各機能の説明 


	
　・メニュー 

	・Preset
	プリセット波形ウィンドウを開きます。

	・Autosave

	"vrc7", "opl", "opn", "opm"の、最後の状態を保存します。
	(もし起動が不安定の場合、このオプションを外してみて下さい。
	リセット状態での起動となります。)


	・Autosave Parameter
	ノブのパラメーターを全て、autosave時へ戻します。


	・Reset Parameter
	ノブのパラメーターを全て、初期状態に戻します。


	・Setting
	環境設定です。各リストボックスに、ファイルドロップで登録します。


	・4op Mask ("PMD"のみ対応します)

	試聴時に、オペレータマスクをします。
	チェックオフで、各オペレータをミュートします。


	試聴時に、SSG-EGを使用します。("PMD 4.8s"以降のみ)

	ドロップダウンメニューで、EGタイプを選択、

	チェックオンで、各オペレータにSSG-EGをアサインします。
	(4.8r以前のPMDでは、Thruを選択して下さい。)


	・オクターブ
	試聴時のオクターブ指定です。


	・FM OP window
	FM Operator ウィンドウを起動します。


	・2op Type
	"nsd", "ppmck:reg", "nsd:reg" の形式で表示できます。

	また、表示形式を切り換えることで、
	各形式に対応したインポート、エクスポートとなります。


	・4op Style

	"pmd", "mucom", "fmp7", "mxdrv" の形式で表示します。
	同じく切り換えることで、インポート、エクスポートとなります。


	・send

	"vrc7"から"opl"変換、"opn"から"opm"変換など、データをエクスポートします。


	・Voice

	クリップボードへTextBoxをコピーします。


	・Advanced

	初期値として読み込まれる、試聴用mmlやheaderファイルを、
	エディタで起動します。

	"mck_header"	"mck_mml"	-- vrc7 mck
	"nsd_header"	"nsd_mml"	-- vrc7 nsd
	"pmd_header"	"pmd_mml"	-- opl,opn,opm


	・Head reload

　	試聴用mml, headerをエディタで修正した際、
	これを即時、反映させるためのものです。

	headerファイルとmmlファイルを、全てリロードします。


 
　・本体パネルの説明 


	キーボードショートカットとして、

	[Space]キーで再生、
	[Enter]キーでも再生します。

	各Text Boxウィンドウにフォーカスがあるときは、

	[Ctrl+ P]で、再生、
	[Ctrl+ O]も、再生、
	[Ctrl+ S]で、インポートとなります。


	・各種ノブ
	各種パラメータを変更します。


	・FMオペレーター選択 / ロール

	Operator を切り換えます。
	切り換えるごとに、各オペレーターのノブを表示します。
　

	・FM音源チップ選択 / ロール
	"vrc7", "opl", "opn", "opm"を選択します。


	・再生ボタン [ > ]

	試聴ボタンです。
	キーボードショートカットでの、[Space][Enter]キーでも可です。


	・インポートボタン [>>]

	音色ボックス、mmlボックスの中身をインポートします。

	各ボックスを、直接エディットした際、
	数値やmmlをインポートします。


	・音色ボックス

	通常、マトリクス音色データが表示されます。

	直接、エディットも可能です。
	最後にインポートボタンを押すことで、内部に読み込まれます。

	外部音色データをインポートする場合、
	このボックスに流し込み、インポートボタンを押して下さい。


	キーボードショートカットは、[Ctrl+ P]で、一時再生します。
	[Ctrl+ S]で、インポートします。


	・mmlボックス

	厳密な音色エディットなどで、
	実際に使用する、mmlを流し込んだりします。

	流し込み後、インポートボタンを押してください。

	キーボードショートカットは、[Ctrl+ P]で、一時再生します。
	[Ctrl+ S]で、インポートします。


 
　・FM Operator ウィンドウ 


	ADSRエンベロープやFM変調波形を、
	視覚的に見せるためのウィンドウです。

	・各Op.Box

	各Boxを[Click]することで、Op.選択をします。

	各Op.での、[ドラッグ移動] or [ホイール] で、
	アサインされた"value"の変更します。

	[右クリック]で、各ノブのアサイン選択をします。


	・BG

	背景を[Click]すると、再生します。
	キーボードショートカットでの、[Space][Enter]キーでも再生します。

	[右クリック]は、オクターブ選択、
	[ホイール]で、アルゴリズムの変更をします。



　・メニュー

	・Property
	FM波形の精度を決めます。

	2/1精度では、サンプルポイント320pointを、縮小表示しています。
	1/2精度では、サンプルポイント80pointで、2px単位の表示となります。

	・Adjust

	Sine波形のグラフ出力調整です。
	音声感覚と視覚のズレを、擦り合わせるためのものです。

	モジュレーションからキャリアへの、モジュレートレベルの変更、
	及び、フィードバックレベルの変更を行います。
	(default値は、20です。)


	・Wait

	Sine波形表示までのウェイトです。
	2/1精度などの負荷が重い時、レスポンスの遅延タイムの指定をします。


 
　・Preset波形 ウィンドウ 


	プリセット波形の再生、及び
	この波形のインポートができます。


	※ -- Setup --

	PMD Preset波形データの読込み手順

	PMD98用 Preset FM音色セットを、
	ダウンロードします。

	[PMD98_PresetVoices.zip]
	"PC88.MML", "X68ED.MML", "EFFEC.MML"

	この3つのファイルを、
	"MC.EXE"があるフォルダに置きます。
	(または、"〜\mml_watch_gui\script\preset"フォルダも可)
　
	File - Preset reload から、各プリセット音色ファイルを読み込みます。


	以後は、プログラム起動時に自動的にロードされます。

	※ -- Setup --


	・リストボックス
	Preset音色リストです。

	リスト内の、@000などを[Click]すると、再生します。
	(右クリックで、音色表示します。)


	・Import ボタン
	インポートボタンを押すと、FM波形エディタへ読み込みます。

 
　・Operator Mask ウィンドウ 


	4op.の"opn", "opm" 時、
	各オペレーターのマスクやSSG-EGを切り換えます。


	・Mask
	チェックoffで、オペレーター出力を停止します。


	・SSG-EG
	チェックonで、SSGエンベロープを使用できます。("opn"のみ可。)

	ドロップダウンメニューから、
	SSGエンベロープフィルターを選択します。(PMD ver.4.8s以降のみ)


	※SSG-EG使用後の解除手続きについて

	SE15,0 (op.1,2,3,4) などを発行する必要があります。
	(op.1であれば、SE1,0が発行される。)

	解除したいop.へチェック入れて、0で再生すると元に戻ります。

  
　　□ Tips 


	
　・楽器ごとのアルゴリズム選択の目安 


　0:	ディストーションギター、チョッパーベース、ハイハット
　1:	ハープ、PSG(programmable sound generator)
　2:	ピアノ、ブラス、エレクトリックギター、ベース、ウッド
　3:	ストリングス、チャイム、フォークギター
　4:	コーラス、フルート、ベル、パスドラム、スネアドラム、タムタム
　5:	ブラス、オルガン
　6:	オルガン、シロホン、ビブラホン、ベースドラム、スネアドラム、タムタム
　7:	パイプオルガン




 
　・2opでのBassの音づくり 


　モジュレーションの、マルチプルを0 (1/2波長)、
　トータルレベルを必要なだけ上げます。(15くらい)


　キャリアのマルチプルを上げて、(3ぐらい)必要となるハイトーンを足します。
　フィードバックで、エレクトリックベース感を足します。(4くらい)


　キャリアで音量を決め、モジュレーションなどで音づくりが基本です。



 
　・4opにおける、3op+ 1opでの楽器波形の再現例 


	アルゴリズム2を使用します。
	主にop2とop3で音色となる倍音成分をつくります。

	op3のトータルレベルは、40とします。
	op2のトータルレベルを上げ、必要となる倍音成分を足します。

	op1とフィードバックを使い、波形全体をねじ曲げ、
	実音のサンプル波形に近づけます。


	モジュレーターのマルチプルの値は、キャリアに対して、

	1,3,5 : 1, 整数倍音(ノコギリ波)、
	2,4,6 : 1, 奇数倍音(矩形波)を意識することで、

	より楽器の音色に近づいていきます。

 
　・既存の音色データを使用した音づくり 


	テキストボックスに、
	使用するFM音色マトリクスを流し込みます。

	この後、[>>]インポートボタンを押すことで読み込みます。


	PMD, FMP7, MUCOM, MXDRV 各形式のインポートに対応します。
	4op styleを選択後、インポートとなります。


	読み込む際、音色データ形式のカウント数をェックします。
	(opnで、42[10*4+2]、opmの場合、46[11*4+2]です)

	正規表現(?<=^|;|:|,|\s)(?:-)?[0-9]+?(?=;|:|,|\s|\n|$)として、
	数値列を読み込む仕様となってます。
	(読み込み失敗は、必要な数値の前にスペースがないケースが多いです。)


	---- インポート例 ----

  opn / 4op style: PMD
  @100   5   5 # フィードバックベース[opn,o2]
    NM ALG  FB
 012 008 007 005 010 014 000 000 000 000
  21, 18, 10, 05, 08, 19, 00, 00, 00, 00,
  24, 12, 11,  5,  7, 20,  0,  3,  0,  0,
  19   6  18   5   7   0   0   5   0   0
  AR  DR  SR  RR  SL  TL  KS  ML  DT AMS

	---- ----

  
　　□ 仕様について 

	音色指定のマイナス値は 0に変換されます。

	("opn", "opm" の、DT1のvalueは、
	0 - 7の範囲で指定して下さい。

	テキストボックス内で、-3 指定を行うと、
	-3 の値は 0に変換されます。)


	op.Boxのサイン波は、
	精度を下げると、プレイヤーの再生波形に比べ歪みが、
	強く表示される傾向があります。


	プレイヤー側に線形補完や、ローパスフィルタが
	入っているケースでは、エディタ側波形出力と幾分違いが出ます。

	(フィードバックがフィルタ発振の状態であっても、
	プレイヤー側で波形補完が入ると、
	波形出力が、穏やかになることが分かってます。

	[1/1 精度のグラフ時、PMDWin.dll側の線形補完を外すと近似値が出る。])



	以下、計算式の概略となります。

	※詳しくは、こちらで解説しています。
	https://eseakisakura.wordpress.com/2022/01/31/fm-powershell-0/


	---- デシベルから電圧・音圧比への変換式

	V2/V1 = 10^(A/20)
	v2= 10^(a/20)* v1
	v2= 10^(-0.75dB* TL/ 20)* v1
	TL: トータルレベル


	--- 直列接続での、FM変調の計算式

	F(t)= A(t)* sin(Wct+ ( I(t)* sinWmt) )
	W: 角速度オメガ
	c:キャリア
	m:モジュレーション


	--- フィードバックの計算式

	F1(t)=  I(t)* sin( Wmt+B F1(t) )
	W: 角速度オメガ
	B: 帰還率ベータ
